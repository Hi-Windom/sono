# v2.0 移动端友好修复算法 Spec

## Why
现有 v1.0/v1.1/v1.2 算法存在三大问题：(1) 在移动端(Termux/ARM)因 96kHz 升采样导致内存溢出和 `resample_poly` 形状不匹配；(2) 各处理模块串行执行，长音频处理耗时过长；(3) 桌面端音质仍有明显提升空间——去削波仅做硬裁剪、降噪门控过于粗暴、谐波增强效率低、响度归一化不精确。v2.0 需要在移动端和桌面端都实现质的飞跃。

## What Changes
- 新增 `backend/services/audio_repair_v2_0.py` — 全新 v2.0 算法模块
- 修改 `backend/services/audio_repair.py` — 注册 v2.0 版本，标记 `mobile_compatible: True`，更新 `DEFAULT_VERSION`
- 修改 `backend/services/audio_repair.py` — 新增参数定义 `warmth`、`clarity`
- 前端无需修改，通过 API 自动获取新版本信息

### v2.0 核心架构变化

**1. 自适应工作采样率（移动端友好）**
- 桌面端：48kHz 全频带处理（不再强制 96kHz，48kHz 已覆盖人耳全频段）
- 移动端：使用原始采样率处理（22.05k/44.1k/48k），避免升采样带来的内存和算力问题
- 仅在用户明确指定更高目标采样率时才升采样，且在最终输出阶段执行

**2. 流式分块处理（内存友好）**
- 固定块大小 2 秒，块间重叠 0.2 秒做交叉淡入淡出
- 每块处理完立即释放内存 `gc.collect()`
- 移动端块大小缩减为 1 秒，进一步降低峰值内存

**3. 处理管线优化**
- 合并同类 STFT 操作：去毛刺 + 去齿音 + 降噪 共享同一次 STFT/ISTFT
- 谐波增强 + 谐波丰富度 合并为一次 STFT
- 减少 STFT/ISTFT 调用次数从 ~8 次降到 ~3 次

**4. 各模块算法升级**

| 模块 | v1.x 问题 | v2.0 改进 |
|------|-----------|-----------|
| 去削波 | v1.2 硬裁剪 `np.clip` | 三次样条插值重建（来自 v1.0/v1.1），保留 v1.1 的边界锚点逻辑 |
| 降噪 | 硬门控截断信号 | 频谱减法 + Wiener 滤波平滑掩码，保留音乐性 |
| 去齿音 | 固定频段阈值 | 自适应频谱质心检测 + 多频段动态压缩 |
| 去毛刺 | 帧能量异常检测 | 帧能量 + 频谱平坦度双指标检测，更精准 |
| 去爆音 | 单尺度检测 | 多尺度能量检测（5ms/10ms/20ms），交叉淡入淡出修复 |
| 谐波增强 | 逐频点遍历 O(n²) | 向量化频段映射 + 基频谐波结构增强 |
| 空间感 | 简单 M/S 增益 | 自适应 M/S + 相关性检测 + 低频居中保护 |
| 响度归一化 | RMS 近似 LUFS | 加权 K-加权响度测量（简化 ITU-R BS.1770） |
| 峰值限制 | 硬裁剪 | 软削波 tanh + lookahead 包络跟随器 |
| 动态范围 | 简单增益调整 | 三频段压缩器（低/中/高）+ 包络跟随器 |

**5. 新增参数**
- `warmth`（温暖度）：控制偶次谐波注入量，让声音更温暖饱满
- `clarity`（清晰度）：控制高频细节增强，让声音更通透

## Impact
- 后端: `backend/services/audio_repair_v2_0.py` (新增), `backend/services/audio_repair.py` (修改)
- 前端: 无需修改，通过 API 自动获取新版本信息
- 移动端: v2.0 标记 `mobile_compatible: True`，在 Termux 环境下可用

## ADDED Requirements

### Requirement: v2.0 修复算法
系统 SHALL 提供 v2.0 版本的音频修复算法，具备以下特性：

#### 特性 1: 自适应工作采样率
- 桌面端（非 MOBILE_MODE）使用 48kHz 工作采样率
- 移动端（MOBILE_MODE）使用原始采样率，不升采样
- 仅在最终输出时按用户指定的 `sample_rate` 参数重采样
- 重采样使用 `resample_poly` 并采用切片赋值避免形状不匹配

#### 特性 2: 流式分块处理
- 桌面端块大小 2 秒，重叠 0.2 秒
- 移动端块大小 1 秒，重叠 0.1 秒
- 块间使用 Hann 窗交叉淡入淡出
- 每块处理完执行 `gc.collect()`

#### 特性 3: STFT 合并优化
- 去毛刺 + 去齿音 + 降噪 共享一次 STFT/ISTFT（频域处理组 A）
- 谐波增强 + 谐波丰富度 共享一次 STFT/ISTFT（频域处理组 B）
- 时域处理（去削波、去爆音、瞬态修复）在频域处理前执行
- 滤波处理（临场增强、低音增强、温暖度、清晰度）在频域处理后执行

#### 特性 4: 频谱减法降噪
- 基于前 N 帧估计噪声频谱
- 使用 Wiener 滤波平滑掩码：`G = max((|S|² - α·|N|²) / |S|², floor)`
- `floor` 参数由 `noise_reduction` 控制，避免音乐噪声
- 掩码时域平滑：相邻帧加权平均

#### 特性 5: 自适应齿音抑制
- 基于频谱质心检测齿音区域（质心突然升高）
- 多频段动态压缩：2-4kHz, 4-8kHz, 8-12kHz
- 压缩量与齿音偏离程度成正比

#### 特性 6: 双指标去毛刺
- 帧能量异常 + 频谱平坦度异常双指标
- 两个指标同时超过阈值才判定为毛刺帧
- 修复方式：邻域频谱插值

#### 特性 7: 三次样条去削波
- 检测超过阈值的削波段
- 使用削波段两侧锚点做三次样条插值重建
- 混合比例由 `de_clipping` 强度控制

#### 特性 8: 多尺度去爆音
- 5ms/10ms/20ms 三个尺度检测能量异常
- 交叉淡入淡出修复，避免咔嗒声
- 修复目标 RMS 取相邻块 RMS 均值

#### 特性 9: 向量化谐波增强
- 基频范围 80-4000Hz，生成 2/3/4 次谐波
- 使用频段映射向量化操作，避免逐频点循环
- 谐波增益随谐波次数递减

#### 特性 10: 自适应空间感
- M/S 编码 + 相关性检测
- 低频（<150Hz）强制居中，避免低频偏移
- Side 增益由相关性和 `spatial_enhance` 强度共同决定

#### 特性 11: K-加权响度归一化
- 简化 ITU-R BS.1770：预滤波（高通 + 高频搁架提升）
- 计算加权 RMS 作为响度估计
- 增益限制在 ±12dB 范围内

#### 特性 12: 软削波峰值限制
- 使用 `tanh` 软削波函数
- lookahead 包络跟随器（5ms attack, 50ms release）
- 阈值 -0.5dB

#### 特性 13: 三频段压缩
- 分频点 250Hz / 4000Hz
- 每频段独立包络跟随器
- 压缩比和阈值由 `dynamic_range` 强度控制

#### 特性 14: 温暖度（偶次谐波注入）
- 对 80-500Hz 基频注入 2 次谐波
- 使用全波整流 + 低通滤波生成偶次谐波
- 混合比例由 `warmth` 控制

#### 特性 15: 清晰度（高频细节增强）
- 对 4-12kHz 频段做轻微高频搁架提升
- 增强量由 `clarity` 控制，最大 +3dB
- 使用二阶 IIR 滤波器实现

### Requirement: v2.0 修复模式
系统 SHALL 提供以下 v2.0 修复模式：

#### 模式 1: AI人声修复
- 针对 AI 人声优化的综合修复
- 重点：去齿音、去毛刺、清晰度增强

#### 模式 2: 深度修复
- 最强修复力度，适合严重损坏的 AI 音频
- 启用所有修复模块的高强度

#### 模式 3: 温和优化
- 轻微处理，保留原始音质特征
- 适合质量较好的 AI 音频微调

#### 模式 4: 专业母带
- 模拟专业母带处理流程
- 重点：响度归一化、动态范围、空间感
