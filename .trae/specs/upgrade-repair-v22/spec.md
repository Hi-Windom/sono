# AI 修复算法 v2.2 升级 Spec

## Why

当前 v2.1 修复算法虽然已在移动端可用，但存在以下问题：
1. **音质问题**：不同类型音乐（人声、器乐、电子、古典）处理效果不够精细
2. **性能瓶颈**：纯 Python/NumPy 实现，在移动端 Termux 环境下处理长音频仍有性能压力
3. **缺乏音乐类型感知**：使用统一参数处理所有类型音乐，无法针对性优化

v2.2 需要在保持移动端兼容性的前提下，通过音乐类型检测和 C 原生库加速，实现更好的音质和性能。

## What Changes

- 新增 `backend/services/repair/repair_v2_2/` 目录 — 全新 v2.2 算法模块
- 新增音乐类型检测模块 — 自动识别人声/器乐/电子/古典等类型
- 集成 C 原生库加速 — 使用 ctypes 调用高性能 DSP 函数
- 针对不同音乐类型的优化处理管线
- 修改 `backend/services/audio_repair.py` — 注册 v2.2 版本

### v2.2 核心架构变化

**1. 音乐类型自动检测**
- 基于频谱特征、节奏特征、谐波结构自动识别音乐类型
- 支持类型：人声主导、纯器乐、电子音乐、古典音乐、流行音乐
- 检测结果影响后续处理参数选择

**2. C 原生库加速（移动端友好）**
- 核心 DSP 运算（STFT/ISTFT、滤波、压缩）使用 C 实现
- 通过 ctypes 调用，无需额外依赖
- 提供纯 Python 回退方案（C 库不可用时）

**3. 类型优化处理管线**

| 音乐类型 | 优化重点 | 特殊处理 |
|---------|---------|---------|
| 人声主导 | 齿音抑制、气息自然度 | 增强去齿音，保护气音 |
| 纯器乐 | 谐波丰富度、空间感 | 增强谐波，优化混响感 |
| 电子音乐 | 低频控制、动态范围 | 收紧低频，增强冲击力 |
| 古典音乐 | 动态保护、细节保留 | 最小化处理，保护动态 |
| 流行音乐 | 平衡处理、响度优化 | 综合优化，适度增强 |

**4. 各模块算法升级**

| 模块 | v2.1 问题 | v2.2 改进 |
|------|-----------|-----------|
| 类型检测 | 无 | 新增智能类型识别 |
| 降噪 | 统一参数 | 按类型调整降噪强度 |
| 去齿音 | 固定频段 | 自适应频段选择 |
| 谐波增强 | 统一增益 | 按类型调整谐波比例 |
| 动态处理 | 固定参数 | 类型自适应压缩 |
| 空间感 | 统一处理 | 按类型调整声场 |
| 响度归一化 | 简单 RMS | 改进的 K-加权算法 |

## Impact

- 后端: `backend/services/repair/repair_v2_2/` (新增目录和模块)
- 后端: `backend/services/audio_repair.py` (修改，注册 v2.2)
- 后端: `backend/services/dsp_native/` (新增 C 原生库封装)
- 移动端: v2.2 标记 `mobile_compatible: True`

## ADDED Requirements

### Requirement: v2.2 修复算法

系统 SHALL 提供 v2.2 版本的音频修复算法，具备以下特性：

#### 特性 1: 音乐类型自动检测
- 基于频谱质心、带宽、零交叉率、谐波结构检测
- 检测置信度 > 0.7 时启用类型优化，否则使用通用处理
- 检测维度：
  - `vocal_ratio`: 人声能量占比
  - `harmonic_complexity`: 谐波复杂度
  - `rhythmic_regularity`: 节奏规律性
  - `spectral_contrast`: 频谱对比度

#### 特性 2: C 原生库加速
- STFT/ISTFT 使用 C 实现（基于 Kiss FFT 或类似）
- 滤波器组使用 C 实现（批量处理多通道）
- 压缩器/限制器使用 C 实现（向量化包络检测）
- 通过 ctypes 加载 `.so` 库（Linux/Android）
- 提供纯 Python 回退实现

#### 特性 3: 类型优化降噪
- 人声：保护 2-4kHz 频段，轻度降噪
- 器乐：全频段均衡降噪
- 电子：强化低频降噪
- 古典：最小降噪，保护细节

#### 特性 4: 自适应去齿音
- 基于频谱质心动态检测齿音区域
- 人声类型：增强 3-8kHz 处理
- 其他类型：轻度处理或跳过

#### 特性 5: 类型优化谐波增强
- 人声：增强 2-3 次谐波，温暖感
- 器乐：增强 2-4 次谐波，丰富度
- 电子：增强低频谐波，冲击力
- 古典：最小谐波增强，保真度优先

#### 特性 6: 智能动态处理
- 人声：轻度压缩，保护动态
- 器乐：中等压缩，增强存在感
- 电子：强力压缩，增强冲击力
- 古典：极轻度压缩，保护动态范围

#### 特性 7: 类型优化空间感
- 人声：适度立体声宽度，聚焦中频
- 器乐：增强空间感，扩展声场
- 电子：强化低频居中，高频扩展
- 古典：保持原始声场，最小处理

#### 特性 8: 改进的响度归一化
- 基于 ITU-R BS.1770-4 简化算法
- 预滤波：高通 + 高频搁架
- 滑动窗口 RMS 测量
- 目标响度 -16 LUFS，容差 ±1 LUFS

### Requirement: v2.2 修复模式

系统 SHALL 提供以下 v2.2 修复模式：

#### 模式 1: 智能修复（默认）
- 自动检测音乐类型
- 应用类型优化的处理参数
- 适合未知来源的音频

#### 模式 2: 人声修复
- 强制使用人声优化参数
- 重点：去齿音、气息自然度、清晰度

#### 模式 3: 器乐修复
- 强制使用器乐优化参数
- 重点：谐波丰富度、空间感、细节保留

#### 模式 4: 深度修复
- 最强修复力度
- 适合严重损坏的 AI 音频

#### 模式 5: 温和优化
- 轻微处理，保留原始音质
- 适合质量较好的音频微调

### Requirement: v2.2 版本注册
