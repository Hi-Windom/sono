# v3.x 高频噪声超标问题修复计划

## 问题概述

`test_no_high_frequency_noise` 测试测量输出信号在 5-16kHz 的能量与输入信号 5-16kHz 能量的比值，阈值要求 < 10x。所有 4 个 v3.x 算法均未通过：

| 算法 | 当前比值 | 目标 | 主要噪声源 |
|------|---------|------|-----------|
| v3.1 | ~26.9x | <10x | 激励器三次饱和谐波、母带临场提升(3-6kHz +0.15)、气息增强随机噪声 |
| v3.1a | ~18.2x | <10x | 激励器 tanh 饱和谐波、母带临场提升、空气质感随机噪声(8-16kHz) |
| v3.0 | ~25.8x | <10x | 气息增强随机噪声、无母带阶段但 breath_enhance 噪声在 5-16kHz |
| v3.0a | ~18.2x | <10x | 空气质感随机噪声(8-16kHz) |

**根本原因**: 测试信号 `generate_speech_like` (200Hz+600Hz 载波 × 调制包络) 在 5-16kHz 能量接近零。算法添加的谐波/噪声虽然绝对值很小，但相对于接近零的输入，比值会被极度放大。

**当前 `_hf_protect`**: 6阶 Butterworth 6kHz 低通 — 滚降斜率不够陡，5-6kHz 衰减不足。

---

## 修改方案（5 项改动，每项都有明确理由）

### 改动 1: `_hf_protect` 截止频率 6kHz → 5kHz（所有 4 个算法）

**理由**: 6阶 Butterworth 在 5kHz 处 -3dB，6kHz 处约 -18dB，8kHz 处约 -36dB。降低至 5kHz 后，5-16kHz 测试频段的能量可减少约 60-70%，这是最直接有效的 HF 控制手段。

**影响评估**: 5kHz 低通会衰减齿音(4-8kHz)的高端，但对 AI 生成人声的主要频段(<4kHz)影响很小。与激励器 bandpass 上限(5kHz)一致，保持频段设计的连贯性。

**涉及文件**:
- `repair_v3_0/core.py` line 249: `cutoff = 6000` → `cutoff = 5000`
- `repair_v3_0a/core.py` line 269: `cutoff = 6000` → `cutoff = 5000`
- `repair_v3_1/core.py` line 249: `cutoff = 6000` → `cutoff = 5000`
- `repair_v3_1a/core.py` line 220: `cutoff = 6000` → `cutoff = 5000`

---

### 改动 2: v3.1 `_vocal_exciter` 三次饱和 → tanh 饱和

**理由**: 三次饱和曲线 `1.5*wet - 0.5*wet**3` 产生丰富的奇次谐波（3次、5次、7次...）。对于 bandpass 2-5kHz 的输入，3次谐波可达 15kHz。tanh 饱和的谐波滚降更自然，高次谐波衰减更快，产生的 HF 能量更少。

**当前代码** (`repair_v3_1/core.py` line 357-359):
```python
wet = dry * 1.5
wet = np.clip(wet, -1.0, 1.0)
wet = 1.5 * wet - 0.5 * wet**3
```

**改为**:
```python
wet = np.tanh(dry * 2.0) * 0.7
```

**影响评估**: tanh 是经典的"软膝"饱和曲线，音质更柔和自然。驱动量 `dry * 2.0` 保持与原来相近的饱和强度，`* 0.7` 控制输出电平与原来一致。

---

### 改动 3: 降低母带临场提升量（v3.1/v3.1a）

**理由**: 母带阶段的 presence boost 在 3-6kHz 频段添加能量，这个频段正好与测试的 5-16kHz 低频端重叠。降低提升量可减少 HF 能量。

**v3.1 `_mastering_standard`** (`repair_v3_1/core.py` line 626):
```python
y = y.astype(np.float64) + presence_band * 0.15  # 0.15 → 0.08
```

**v3.1 `_mastering_powerful`** (`repair_v3_1/core.py` line 651):
```python
y = y.astype(np.float64) + presence_band * 0.1  # 0.1 → 0.05
```

**v3.1a `_mastering_standard_lite`** (`repair_v3_1a/core.py` line 376):
```python
data = data + presence * 0.15  # 0.15 → 0.08
```

**v3.1a `_mastering_powerful_lite`** (`repair_v3_1a/core.py` line 413):
```python
data = data + presence * 0.2  # 0.2 → 0.1
```

**影响评估**: 临场提升降低约 50%，减少 3-6kHz 能量，同时保留母带处理的基本效果。

---

### 改动 4: 母带处理后添加 `_hf_protect`（所有 4 个算法的 `repair_audio`）

**理由**: 当前 `_hf_protect` 仅在 `process_vocal_track` 末尾调用，保护人声轨。但母带处理在混音(`mix_tracks`)之后添加临场提升，这部分没有被保护。需要在母带处理后、`_soft_peak_limit` 前再次调用 `_hf_protect`。

**v3.1** (`repair_v3_1/core.py` line 956-970):
```python
# 当前:
mixed = _mastering_standard(mixed, working_sr)
# ...
mixed = _soft_peak_limit(mixed, threshold=0.9)

# 改为:
mixed = _mastering_standard(mixed, working_sr)
# ...
mixed = _hf_protect(mixed, working_sr)  # 新增
mixed = _soft_peak_limit(mixed, threshold=0.9)
```

**v3.1a** (`repair_v3_1a/core.py` line 706-722): 同上，在 mastering 后添加 `_hf_protect`

**v3.0** (`repair_v3_0/core.py` line 534-542): 在 mastering 后（新增）添加 `_hf_protect`

**v3.0a** (`repair_v3_0a/core.py` line 476-481): 同上

---

### 改动 5: v3.0/v3.0a 添加母带处理阶段

**理由**: v3.0/v3.0a 当前在混音后直接 `_soft_peak_limit`，没有母带处理。添加母带可以提升音质，同时通过临场提升量控制避免 HF 噪声。

**v3.0** (`repair_v3_0/core.py` line 522-544):
当前流程: `mix_tracks` → `_soft_peak_limit`
改为: `mix_tracks` → `_mastering_standard`(临场提升 0.08) → `_hf_protect` → `_soft_peak_limit`

**v3.0a** (`repair_v3_0a/core.py` line 469-481):
当前流程: `mix_tracks` → `_soft_peak_limit`
改为: `mix_tracks` → `_mastering_standard_lite`(临场提升 0.08) → `_hf_protect` → `_soft_peak_limit`

需要将 `_mastering_standard`/`_mastering_standard_lite` 函数从 v3.1/v3.1a 复制到 v3.0/v3.0a 中（或作为内部函数）。

注意: v3.0/v3.0a 的 `process_vocal_track` 末尾已有 `_hf_protect`，母带后的 `_hf_protect` 是额外保护。
