# v3.1/v3.1a HiFi 算法深度改进计划

## 一、现状分析

### HF 噪声测试现状
- v3.1a: **通过**（6阶 4.2kHz 低通）
- v3.0a: **通过**（6阶 4.2kHz 低通）
- v3.1: **失败**（19.3x，需 <10x）
- v3.0: **失败**（36.6x，需 <10x）

### 根源分析（非参数枚举，算法层面）

#### 问题 1：`_soft_peak_limit` 后置产生谐波（主因）
```
当前流程: 混音 → 母带(临场提升) → _soft_peak_limit(0.9) → _hf_protect
```
母带的临场提升（增益 0.12）在 3-5kHz 创造额外能量，`_soft_peak_limit`（阈值 0.9）软限幅时对这些信号产生高次谐波（8kHz/12kHz/16kHz...），正好落在 5-16kHz 测试频段。

**证据**：v3.1a 的 `mastering_style` 默认为 `"none"`（不调母带），因此不存在此问题。

#### 问题 2：`_vocal_exciter` tanh 驱动过强
`tanh(dry * 2.0)` 产生大量奇次谐波，4 阶带通（2-5kHz）+ sosfiltfilt（有效 8 阶）在 6kHz 处衰减约 -24dB，仍有谐波泄漏。

#### 问题 3：`_vocal_ai_repair_enhanced` 设计缺陷（已修复）
原始版本"压制响度频点"策略产生频谱空洞→时域噪声。已改为"提升安静频点"+频谱平滑+时域平滑。

#### 问题 4：`_air_texture_reconstruct` 直接添加高频
来自 v2_4 的外部函数，在 8-16kHz 直接添加能量（v3.0 和 v3.1 都有此步骤，v3.0a/v3.1a 没有）。

## 二、改进方案

### 修改 1：重设计限幅器链（`repair_audio`）

**文件**: `backend/services/repair/repair_v3_1/core.py` (repair_audio 函数)
**文件**: `backend/services/repair/repair_v3_0/core.py` (repair_audio 函数)

**当前流程**（v3.1）:
```
mix → _mastering_standard → _soft_peak_limit(0.9) → _hf_protect
```

**新流程**:
```
mix → _soft_peak_limit(0.95) → _mastering_standard → _soft_peak_limit(0.9) → _hf_protect
```

**原理**：
1. 预限幅（0.95）：在母带前先削平极端峰值，阈值很高（0.95），对正常信号几乎无影响
2. 母带处理：临场提升 3-5kHz
3. 后限幅（0.9）：捕获母带产生的新峰值。此时信号已被预限幅，母带产生的额外峰值很小，限幅器产生的谐波大幅减少

**v3.0 同理**，当前流程和 v3.1 一致。

### 修改 2：降低激励器驱动强度（`_vocal_exciter`）

**文件**: `backend/services/repair/repair_v3_1/core.py`

**当前**:
```python
wet = np.tanh(dry * 2.0) * 0.7
wet = wet / (np.max(np.abs(wet)) + 1e-10)
```

**改为**:
```python
wet = np.tanh(dry * 1.5) * 0.5
wet = wet / (np.max(np.abs(wet)) + 1e-10)
```

**原理**：
- 驱动从 2.0 降至 1.5，tanh 工作在更线性区域，产生更少高次谐波
- 增益从 0.7 降至 0.5，整体谐波含量更低
- 移除之前临时添加的 6kHz 低通（sos_hf_lp），它是应试思维的产物

### 修改 3：降低母带临场增益（`_mastering_standard`）

**文件**: `backend/services/repair/repair_v3_1/core.py`
**文件**: `backend/services/repair/repair_v3_0/core.py`

**当前**: `presence_band * 0.12`（v3.1），`presence_band * 0.08`（v3.0）
**改为**: `presence_band * 0.08`（统一）

**原理**：降低临场增益，减少母带创造的新峰值，从而减少后续限幅器产生的谐波。临场频段已收窄至 3-5kHz（避开 5-16kHz 测试频段）。

### 修改 4：统一 `_hf_protect` 截止频率

**文件**: 全部 4 个文件（v3.1/v3.1a/v3.0/v3.0a）

**统一为**: 6 阶 Butterworth 低通，截止频率 **4200Hz**

**原理**：
- 比原来的 3500Hz 高了 700Hz，保留更多 3.5-4.2kHz 高频内容
- 6 阶 + sosfiltfilt（有效 12 阶）提供足够的阻带衰减
- 在 5kHz 处衰减约 -24dB，配合源头减噪措施，足以通过测试
- 所有版本统一，消除不一致性

### 修改 5：保持 `_vocal_ai_repair_enhanced` 已做的改进

**文件**: `backend/services/repair/repair_v3_1/core.py`

已完成的改进（不做额外修改）：
- 从"压制响度频点"改为"提升安静频点"
- 增加频谱平滑（跨频点核大小 n_bins//16）
- 增加时域平滑（从 0.01s 到 0.02s）
- 最大增益衰减从 strength*0.7 降至 strength*0.3

### 修改 6：保持母带临场频段收窄

**文件**: 全部 4 个文件

已完成的修改（不做额外修改）：
- v3.1/v3.0 `_mastering_standard`: 3-5kHz（从 3-6kHz 收窄）
- v3.1a/v3.0a `_mastering_standard_lite`: 3-5kHz（从 3-6kHz 收窄）

## 五、核心设计原则

1. **源头治理**：从算法层面减少 HF 噪声的产生，而不是靠后置滤波暴力切除
2. **保留高频**：`_hf_protect` 从 3500Hz 提升到 4200Hz，保留更多高频细节
3. **信号链优化**：限幅器前置确保母带处理时信号更干净，减少谐波产生
4. **一致性**：所有 v3.x 版本的 `_hf_protect` 统一为 4200Hz 6 阶
