# 音频质量异常调查计划

## 问题描述
用户报告交付的 WAV 和 MP3 音质均不正常，特别是 MP3 直接变成"怪物低语"。单轨双轨均有问题。

## 调查步骤

### 1. 确认问题范围
- [ ] 确认 WAV 和 MP3 是否都有问题
- [ ] 确认单轨和双轨是否都有问题
- [ ] 确认问题出现的时间点（什么时候开始出现的）

### 2. 检查最近的代码变更
需要调查最近修改的文件：
- `backend/services/mp3_encoder.py` - MP3 编码逻辑
- `backend/api/routes.py` - 下载端点和渲染逻辑
- `backend/database.py` - 缓存查找逻辑
- `backend/services/render.py` - 音频渲染输出
- `backend/services/audio_loader.py` - 音频加载

### 3. 重点怀疑对象
根据"怪物低语"的描述，这通常是：
- 采样率不匹配（音频被错误地以不同采样率播放）
- 数据格式转换问题（float/int16 转换错误）
- 缓冲区问题（数据被损坏或截断）

### 4. 具体检查点
- [ ] 检查 mp3_encoder.py 中的采样率读取和设置
- [ ] 检查 WAV 导出时的采样率和数据格式
- [ ] 检查 render.py 中的重采样逻辑
- [ ] 检查 audio_loader.py 的音频加载格式
- [ ] 对比 git diff 查看具体变更

### 5. 测试验证
- [ ] 创建测试用例验证音频质量
- [ ] 对比修复前后的输出
- [ ] 确认问题是否在特定条件下出现

## 假设
1. 问题可能出现在最近对 MP3 编码器的修改中
2. 可能影响到了 WAV 导出或音频处理流程
3. 需要确认是否有未提交的代码变更引入了 bug

## 下一步行动
1. 首先询问用户具体症状和复现步骤
2. 检查代码变更历史
3. 创建测试验证音频质量
4. 定位并修复问题
