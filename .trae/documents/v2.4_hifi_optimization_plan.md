# v2.4 HiFi 优化计划

## 1. 当前问题分析

### 1.1 v2.4 与 v2.4a 核心差异

| 维度 | v2.4 (桌面版) | v2.4a (移动版) |
|------|---------------|----------------|
| 处理步骤 | 14+ 步 | ~10 步 |
| 音乐类型检测 | 完整 MusicTypeDetector | 快速 _detect_music_type_fast |
| AI频谱修复 | 复杂 _ai_artifact_repair | 简化 _ai_artifact_repair_lite |
| 动态处理 | 多段压缩 (三分频) | 全局压缩 |
| 低频增强 | 谐波次低频+激励+body | 简化次谐波 |
| 空气质感 | 谐波重建+噪声注入 | 简化噪声注入 |
| 空间处理 | spatial_enhance_v6, stereo_width_v3 | 无 |
| 降噪 | spectral_group_a + noisereduce | 简化 _spectral_denoise |

### 1.2 为什么 v2.4 听感不如预期

1. **过度处理**: 14+ 步骤累积引入处理伪影
2. **多段压缩副作用**: 三分频处理破坏相位一致性，导致"糊"
3. **AI修复随机抖动**: `jitter_db` 引入不必要的噪声
4. **HiFi模式只是降参数**: 未针对高保真优化算法本身
5. **缺少节奏感知**: 无法区分 BPM 高低进行差异化处理

## 2. 优化目标

基于用户反馈，v2.4 应该：
- ✅ 更好的**细节保留**（高频、空气感）
- ✅ **更低的处理伪影**（透明感）
- ✅ **更好的动态表现**（音乐性压缩）
- ✅ **BPM自适应**: 高BPM更"有力""不糊"，低BPM更"通透""华丽"

## 3. 优化方案

### 3.1 新增 v2.4 专属模块

#### 3.1.1 节奏分析器 (TempoAnalyzer)
**文件**: `backend/services/repair/repair_v2_4/tempo_analyzer.py`

**功能**:
- 检测 BPM（每分钟节拍数）
- 计算节奏密度（onset density）
- 输出节奏特征：快/中/慢 + 密度高/中/低

**算法**:
```python
- 使用 onset detection（基于频谱差分）
- 通过自相关或峰值间隔估计 BPM
- 分类阈值：
  - 快节奏: BPM > 120
  - 中节奏: 80 <= BPM <= 120
  - 慢节奏: BPM < 80
```

#### 3.1.2 HiFi 多段压缩器 (HifiMultibandCompressor)
**文件**: `backend/services/repair/repair_v2_4/hifi_multiband.py`

**改进点**:
- 使用线性相位分频（保持相位一致性）
- 自适应分频点（基于音乐类型）
- 音乐性压缩曲线（更柔和的 knee）
- 可选：动态分频（根据频谱内容调整）

**BPM自适应策略**:
```python
if BPM > 120 (快节奏):
    - 更快的 attack (保留瞬态冲击力)
    - 更短的 release (避免糊)
    - 轻微增加 presence (2-5kHz)
elif BPM < 80 (慢节奏):
    - 更慢的 attack (更自然)
    - 更长的 release (更平滑)
    - 增强空气感 (10kHz+)
    - 轻微增加 stereo width
```

#### 3.1.3 HiFi AI 频谱修复 (HifiAIArtifactRepair)
**文件**: `backend/services/repair/repair_v2_4/hifi_ai_repair.py`

**改进点**:
- 移除随机抖动 (`jitter_db`)
- 使用更保守的平滑策略
- 保护瞬态（transient preservation）
- 频谱掩蔽（spectral masking）避免过度处理

#### 3.1.4 自适应细节增强 (AdaptiveDetailEnhance)
**文件**: `backend/services/repair/repair_v2_4/detail_enhance.py`

**功能**:
- 基于BPM的细节增强策略
- 高BPM: 强调瞬态清晰度，轻微高频提升
- 低BPM: 强调空间感和空气感
- 使用并行处理（dry/wet mix）保持自然度

### 3.2 修改 v2.4 核心文件

#### 3.2.1 `backend/services/repair/repair_v2_4/core.py`

**修改内容**:

1. **导入新模块**:
```python
from .tempo_analyzer import TempoAnalyzer, get_tempo_params
from .hifi_multiband import HifiMultibandCompressor
from .hifi_ai_repair import hifi_ai_artifact_repair
from .detail_enhance import AdaptiveDetailEnhance
```

2. **新增处理流程**（v2.4 专属）:
```python
# 在 repair_audio 函数中

# 1. 节奏分析（v2.4 专属）
tempo_info = TempoAnalyzer().analyze(y, sr)
tempo_params = get_tempo_params(tempo_info)

# 2. 使用 HiFi 多段压缩替代原有 _enhanced_multiband_compress
if params.get("dynamic_range", 0) > 0:
    compressor = HifiMultibandCompressor(sr, music_type, tempo_params)
    y = compressor.process(y, params["dynamic_range"])

# 3. 使用 HiFi AI 修复替代原有 _ai_artifact_repair
if params.get("ai_repair", 0) > 0:
    y = hifi_ai_artifact_repair(y, sr, params["ai_repair"], tempo_params)

# 4. 自适应细节增强（v2.4 专属新步骤）
if not is_hifi:  # 标准模式下也启用
    detail_enhancer = AdaptiveDetailEnhance(sr, tempo_params)
    y = detail_enhancer.process(y, amount=0.3)
```

3. **优化 HiFi 模式**:
```python
if is_hifi:
    # 不再只是简单降参数，而是启用专门的 HiFi 处理路径
    # 跳过可能引入伪影的步骤
    # 使用更高精度的处理
```

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| BPM检测不准确 | 自适应策略失效 | 添加置信度阈值，低置信度时使用默认参数 |
| 处理时间增加过多 | 用户体验下降 | 优化算法，使用更高效的实现 |
| 主观听感提升不明显 | 优化无效 | 增加A/B测试迭代，调整参数 |
| 与现有功能冲突 | 回归问题 | 保持原有代码路径，新增代码独立 |
