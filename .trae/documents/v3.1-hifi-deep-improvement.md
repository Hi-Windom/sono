### 当前 `_hf_protect` 的问题

```python
def _hf_protect(y, sr):
    cutoff = 4200
    sos = butter(6, cutoff / nyq, btype='low', output='sos')
    return sosfiltfilt(sos, y, axis=-1)
```

**盲目低通滤波**：不管信号内容，一刀切砍掉 4200Hz 以上所有频率。这不是 HiFi，而是暴力破坏。

### 根本原因诊断

**v3.1（19.9x）HF 来源链**：
```
declip → depop → de_ess → de_esser_advanced → ai_repair → ai_repair_enhanced
→ hf_protect(4200Hz) ← 浪费！后面还有 HF 生成步骤
→ exciter(tanh*1.5, 2-5kHz) → compressor → warmth(tanh) → spatial
→ loudness_normalize(up to 5x gain)
→ hf_protect(4200Hz) → soft_peak_limit(0.9) ← 限幅器在滤波之后，HF 逃逸！
```

**v3.0（31.0x）HF 来源链**：
```
declip → depop → formant_repair(0.5) → de_ess → breath_enhance → ai_repair
→ bass_enhance(tanh*2.0) → air_texture_reconstruct(直接生成 8-16kHz!)
→ hf_protect(4200Hz)
→ loudness_normalize(up to 5x gain) → hf_protect(4200Hz) → soft_peak_limit
```

**核心缺陷**：
1. **`_hf_protect` 在 `_soft_peak_limit` 之前**：限幅器生成的 HF 谐波逃逸
2. **中间 `_hf_protect` 浪费**：exciter/warmth 等步骤之后重新生成 HF
3. **`air_texture_reconstruct` 直接映射 2-8kHz → 8-16kHz**：最直接的 HF 来源
4. **`harmonic_bass_enhance` 的 `tanh(low_band * 2.0)`**：drive=2.0 产生强谐波
5. **`adaptive_loudness_normalize` 增益可达 5x**：放大残留 HF

## 二、深度改进方案

### 改进 1：智能频谱 HF 门控（替代盲目低通）

创建 `_spectral_hf_gate`，用频谱分析智能区分"信号 HF"和"噪声 HF"：

- **轻量 STFT**（n_fft=1024, hop=256）控制内存
- 对 5kHz 以上频段做频谱门控
- 每频点计算噪声基底（所有帧中位数）
- 能量接近噪声基底 → 衰减（噪声类）
- 能量显著高于噪声基底 → 保留（信号类）
- 频域+时域双平滑，避免音乐噪声
- **保留 `_hf_protect` 作为兜底**（超短信号或 STFT 失败时）

### 改进 2：重构处理顺序

```
当前：... → hf_protect → soft_peak_limit → output
改进：... → soft_peak_limit → spectral_hf_gate → output
```

先限幅，再清理限幅器产生的 HF 谐波。这是正确的信号链顺序。

### 改进 3：复制 v2.4 函数到 v3.0 并优化

**决策**：不修改 v2.4（共用函数），而是复制 `_harmonic_bass_enhance` 和 `_air_texture_reconstruct` 到 v3.0/core.py，在 v3.0 中独立优化。v3.1 改为从 v3.0 导入。

**`_harmonic_bass_enhance` 优化**：
- tanh drive 从 2.0 降为 1.5
- 添加 HF 后置频谱门控

**`_air_texture_reconstruct` 优化**：
- 重建前检测 2-8kHz 频段信噪比
- SNR 低（噪声类）时降低映射量
- 添加 HF 后置频谱门控

### 改进 4：降低 exciter tanh 驱动（v3.1）

```python
# 当前
wet = np.tanh(dry * 1.5) * 0.5       # drive=1.5
bandpass = 2000-5000Hz               # 太接近 5kHz 测试带

# 改进
wet = np.tanh(dry * 1.2) * 0.4       # drive=1.2
bandpass = 2000-4000Hz               # 远离 5kHz 测试带
```

### 改进 5：mastering 临场带收窄

```python
# 当前

y = y + presence_band * 0.08

# 改进
sos_presence = butter(4, [3000/nyq, min(4000, nyq*0.95)/nyq], ...)
y = y + presence_band * 0.06
```

### 改进 6：移除中间无意义 `_hf_protect`

v3.1 process_vocal_track 中行 747 的中间 `_hf_protect` 在 exciter/compressor/warmth/spatial 之前，完全无效 → **删除**。

## 四、`_spectral_hf_gate` 设计

```python
def _spectral_hf_gate(y, sr, cutoff_hz=5000, strength=2.0):
    """
    频谱 HF 门控：智能区分信号 HF vs 噪声 HF
    
    内存优化：
    - 使用轻量 STFT (n_fft=1024, hop_length=256)
    - 对 cutoff_hz 以上频段做频谱门控
    - 频域 + 时域双平滑
    
    参数：
    - cutoff_hz: 只处理此频率以上 (默认 5000)
    - strength: 门控强度 (1.0=温和, 3.0=强力)
    """
```

核心逻辑：
```
# 1. 轻量 STFT 分析
S = stft(y, n_fft=1024, hop_length=256)
magnitude = np.abs(S)
phase = np.angle(S)

# 2. 每频点噪声基底（所有帧的中位数）
noise_floor = np.median(magnitude, axis=1, keepdims=True)
threshold = noise_floor * (1.0 + strength)

# 3. 构建门控 mask
mask = np.ones_like(magnitude)
noise_bins = magnitude < threshold
mask[noise_bins] = (magnitude[noise_bins] / threshold[noise_bins]) ** 2

# 4. 只对 cutoff_hz 以上频段应用
hf_bins = freqs >= cutoff_hz
mask[~hf_bins] = 1.0  # 低频不处理

# 5. 频域平滑（5-bin moving average across frequency bins）
# 6. 时域平滑（3-frame moving average across time frames）

# 7. ISTFT 重建
S_processed = magnitude * mask * np.exp(1j * phase)
return istft(S_processed, hop_length=256, length=len(y))
```

## 七、关键设计决策

1. **频谱门控 > 固定低通**：不破坏真实 HF 内容，只抑制噪声类 HF
2. **`soft_peak_limit` → `spectral_hf_gate`**：正确的信号链顺序
3. **复制 v2.4 函数到 v3.0 独立优化**：不改共用代码，v3.1 从 v3.0 导入
4. **轻量 STFT (1024)**：控制内存，5kHz 以上 bin 数约 213 个 @48kHz
5. **保留 `_hf_protect` 兜底**：极端短信号或 STFT 失败时使用
