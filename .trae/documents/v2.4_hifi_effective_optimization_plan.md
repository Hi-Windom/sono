# v2.4 HiFi 确实有效的优化方案

## 问题根因分析

通过对比 v2.4 和 v2.4a 的完整处理流程，我发现了**关键问题**：

### 1. 处理顺序严重错误（最严重的问题）
**v2.4a（表现好）**：
```
削波 → 爆音 → 降噪 → 齿音 → AI修复 → 响度归一化 → 压缩 → 低频 → 空气感
```

**v2.4（表现差）**：
```
削波 → 爆音 → 瞬态 → 响度归一化 → 压缩 → AI修复 → 降噪 → 谐波 → 空间 → 低频 → 空气感 → 细节增强
```

**问题**：
- `响度归一化` 放在 `压缩器` 之前 → 压缩器工作电平不一致
- `降噪` 放在 `压缩` 之后 → 压缩后的噪声被放大
- 处理步骤过多 → 过度处理引入伪影

### 2. 流程过于复杂
- v2.4: 14+ 步处理
- v2.4a: ~10 步处理
- 很多步骤互相抵消效果

### 3. 新增模块未真正发挥作用
- HiFi 多段压缩：只是替换函数，但没有优化处理顺序
- 自适应细节增强：放在最后，前面处理已经破坏细节

## 优化方案

### 策略：v2.4 应该是 v2.4a 的「HiFi增强版」，不是完全重构

保持 v2.4a 的**优秀流程**，在此基础上添加 v2.4 的**高级功能**，而不是重新搞一套。

### 优化后的流程

```
[保持 v2.4a 的顺序，插入 v2.4 的高级功能]
1. 削波修复
2. 爆音修复
3. 节奏分析（新增）
4. 频谱降噪
5. 齿音抑制
6. HiFi AI频谱修复（替换，带瞬态保护）
7. 响度归一化
8. HiFi 多段压缩（替换，BPM自适应）
9. 低频增强（可选）
10. 空气质感重建（可选）
11. 自适应细节增强（新增）
12. 立体声增强（BPM自适应）
13. 峰值限制
```

### 具体改动

#### 1. 重构 v2.4 的 `repair_audio` 函数
- 采用 v2.4a 的核心流程
- 保持处理顺序：**先修复，再归一化，最后增强**
- 只保留真正有效的模块

#### 2. 简化参数配置
- 降低默认强度，避免过度处理
- 移除相互冲突的参数

#### 3. 优化 HiFi 模块的应用方式
- 节奏分析放在前面，影响后续处理
- 自适应细节增强放在合适的位置（在主要修复之后）
- 立体声增强根据 BPM 自适应

#### 4. 保留但精简 v2.4 之前的高级功能
- 空间增强、温暖度等可选，但默认关闭
- 避免过度处理

## 预期效果

- **处理顺序正确**：先修复，再归一化，最后增强
- **步骤减少**：从 14+ 步降到 ~12 步，但更有效
- **HiFi特性保留**：节奏感知、BPM自适应、高级压缩
- **与 v2.4a 明显差异**：
  - 快节奏：更有力、更清晰
  - 慢节奏：更通透、更华丽
  - 整体更自然、细节更好
