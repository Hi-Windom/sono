# v2.2 音质优化计划

## 用户反馈
- 问题：降噪过度/失真、压缩过度/动态损失、听感不够 hifi
- 音频类型：音乐为主
- 优化策略：算法改进

## 当前状态分析

### 1. 降噪算法问题
当前使用频谱减法降噪：
```python
G = np.maximum((mag ** 2 - alpha * noise_profile ** 2) / (mag ** 2 + 1e-10), floor)
```
问题：频谱减法容易引入"音乐噪声"(musical noise)，听起来不自然。

### 2. 压缩算法问题
当前使用多段压缩：
- 人声：threshold -20dB, ratio 2.5-3.0
- 电子：threshold -16dB, ratio 3.0-4.0
- 古典：threshold -26dB, ratio 1.2-2.0

问题：压缩阈值过低、比例过高，导致动态范围损失。

### 3. 整体处理链问题
- 处理步骤过多（13+ 步），累积失真
- 部分模块参数过强
- 缺乏心理声学优化

## 优化方案

### Phase 1: 改进降噪算法
**文件**: `spectral_group_a.py`

将频谱减法改为 **Wiener 滤波** + **心理声学掩蔽**：
1. 使用更平滑的噪声估计
2. 引入人耳听觉掩蔽模型
3. 保留音乐瞬态细节

### Phase 2: 优化压缩参数
**文件**: `type_params.py`, `dynamics.py`

调整压缩参数：
- 提高阈值（减少压缩量）
- 降低压缩比例
- 增加 attack 时间，保护瞬态

### Phase 3: 改进谐波增强
**文件**: `spectral_group_b.py`

优化谐波生成算法：
- 使用更自然的谐波混合
- 避免过度增强高频

### Phase 4: 优化处理流程
**文件**: `core.py`

- 减少不必要的处理步骤
- 调整处理顺序（先动态后频谱）
- 添加处理质量选择（标准/高质量）

### Phase 5: 改进响度归一化
**文件**: `postprocess.py`

- 使用更精确的 LUFS 测量
- 添加 True Peak 限制

## 验证标准
1. 降噪后无明显音乐噪声
2. 压缩后动态范围保留 > 80%
3. 整体 THD+N < 0.1%
4. 主观听感：自然、透明、不疲劳
