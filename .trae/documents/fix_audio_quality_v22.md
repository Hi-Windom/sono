# 修复 v2.2/v2.2a 音质问题

## 问题总结

用户反馈：
- **v2.2 桌面端**: "完全修坏了" — 音量异常、频谱失真、动态 flattened
- **v2.2a 移动端**: "频繁破音" — 削波/爆音修复过度或压缩器增益过大

## 根本原因分析

### v2.2 桌面端问题

1. **子带处理破坏频谱完整性** (`subband_processing.py`)
   - QMF 滤波器组不是完美重构，子带相加引入相位失真和频谱泄漏
   - 每个子带独立做 STFT/ISTFT，多次变换累积误差
   - 中频子带合并 `subbands[1] + subbands[2]` 后再处理，破坏原始相位关系

2. **多段压缩过度** (`dynamics.py`)
   - `makeup_gain = 10 ** (makeup / 20)`  makeup 最大 6dB，累积增益过大
   - 三频段分别压缩后相加，可能产生梳状滤波效应
   - 未限制最大增益，导致某些频段被过度提升

3. **响度归一化过度增益** (`postprocess.py`)
   - `gain_db = np.clip(target_lufs - current_lufs, -12, 12)` — 最大 +12dB 增益
   - 对已经过压缩的音频再施加 +12dB 增益，必然削波
   - 随后峰值限制器强行压回，产生失真

4. **处理顺序问题** (`core.py`)
   - 动态压缩在频谱处理之前，但响度归一化在最后
   - 压缩 → 频谱处理 → 音色调整 → 响度归一化，每一步都可能改变动态

### v2.2a 移动端问题

1. **单段压缩器 makeup gain 过大** (`core.py`)
   - `makeup_gain = 1.0 + (1.0 - 1.0/ratio) * amount * 0.3`
   - ratio=4, amount=0.3 时 makeup=1.225 (+1.8dB)
   - 分块包络检测导致增益突变，产生" pumping "

2. **硬限幅导致破音** (`_peak_limit`)
   - 简单 `y * (threshold/peak)` 全局缩放
   - 如果前面步骤已经产生削波，硬限幅产生谐波失真

3. **频谱减法过度** (`_spectral_repair`)
   - `gain = np.maximum(gain, 0.15)` — 最小增益 0.15，过度衰减
   - 时间平滑 `alpha=0.7` 不够，产生"音乐噪声"

## 修复方案

### 1. 修复 v2.2 桌面端

#### 1.1 移除子带处理（临时方案）
- 将 `subband_processing.py` 从核心流程中移除
- 回退到传统的 `spectral_group_a` + `spectral_group_b` 处理
- 子带处理作为实验性功能，默认关闭

#### 1.2 修复多段压缩
- 限制 makeup gain 最大 +3dB
- 压缩后添加安全限幅
- 调整频段分割参数，避免过度重叠

#### 1.3 修复响度归一化
- 限制最大增益为 +6dB（而非 +12dB）
- 添加预压缩检测：如果输入已经过压缩，降低增益
- 使用更长的测量窗口（3秒而非400ms），避免对瞬态过度反应

#### 1.4 修复处理顺序
- 响度归一化移到压缩之前
- 最终只保留一个峰值限制器
- 顺序：削波修复 → 爆音修复 → 响度归一化 → 频谱处理 → 动态压缩 → 峰值限制

### 2. 修复 v2.2a 移动端

#### 2.1 修复单段压缩
- 降低 makeup gain 系数
- 添加增益平滑，避免突变
- 限制最大增益为 +3dB

#### 2.2 修复硬限幅
- 使用软限幅替代硬限幅
- 添加 look-ahead 限幅器

#### 2.3 修复频谱减法
- 提高最小增益 floor 到 0.3
- 增加时间平滑系数到 0.9
- 使用更保守的噪声估计
